# Datasheet — SEBO NeurIPS BBO Query Dataset

**Author:** Nikolas Karefyllidis, PhD
**Last updated:** June 2026

---

## Motivation

**Purpose.** This dataset records the query-response history from applying SEBO to the NeurIPS 2020 Black-Box Optimisation Challenge format — 8 unknown objective functions, one evaluation per function per round, 13 rounds. It supports reproducibility of the BO pipeline, benchmarking of acquisition strategies, and portfolio demonstration.

**Status.** Collection is complete (13 rounds, frozen). No further oracle evaluations are scheduled.

## Collection Process

Queries were generated by SEBO's GP-based pipeline in per-function Jupyter notebooks (`notebooks/function_N_*.ipynb`):

1. Fit three GP kernels (RBF, Matérn ν=1.5, RBF+WhiteKernel) to all prior observations.
2. Select the best kernel by log-marginal likelihood (LML).
3. Optimise EI, PI, or UCB over a Sobol/LHS candidate set of 2^15–2^18 points.
4. Return the candidate with the highest acquisition value, subject to minimum-distance and boundary constraints.

The first 10 observations per function were provided as initial warm-start data (not generated by SEBO). Subsequent observations were generated by the pipeline and appended via `append_results/append_weekN_results.py`.

**Sampling strategy.** Sequential model-based: each query is conditioned on all prior (x, y) pairs. Early rounds used higher exploration weighting (UCB with κ = 2–3, EI with ξ = 0.05–0.1). Later rounds shifted toward exploitation (EI with ξ = 0.01, UCB with κ = 1–2) as the surrogate became confident near promising regions.

## Preprocessing

**Output warping.** Applied before GP fitting for three functions with skewed or multi-scale y:
- F1: log transform — y mostly near zero with a single large value.
- F5: log transform — y spans four orders of magnitude [0.11, 7493.9].
- F7: log transform — y spans [0.003, 2.797].
- F2, F3, F4, F6, F8: no transform.

The inverse transform is applied when reporting results; raw y is always stored in `observations.csv`.

**Duplicate-x handling.** Observations at the same x (as occurred for F6) are retained via idempotent append logic — oracle-canonical values are preserved for reproducibility and noise modelling.

## Uses

**Intended:**
- Training and evaluating GP surrogates for sequential black-box optimisation.
- Benchmarking acquisition functions (EI, PI, UCB, ensemble) on real oracle feedback.
- Demonstrating Bayesian optimisation workflows, exploration–exploitation trade-offs, and surrogate diagnostics.

**Limitations for other uses.** The dataset is small (20–51 points per function) and domain-specific. The query distribution is biased toward regions promising under this specific GP/acquisition strategy, not toward uniform coverage.

**Inappropriate uses.**
- Do not infer the true functional form of the oracle — functions are unknown and observations are too sparse.
- Do not use one function's observations as training data for a surrogate of a different function.
